# xiaoli（小李）—— 技术架构设计

> 本文档定义 xiaoli 的技术实现方案。

---

## 1. 技术选型

| 层 | 选型 | 理由 |
|---|---|---|
| CLI 框架 | Node.js + TypeScript | npm 生态、跨平台、Claude Code 同栈 |
| 包管理 | npm（全局安装） | `npm install -g xiaoli` |
| AI SDK | DeepSeek API（OpenAI 兼容） | 国内可用、性价比、用户偏好 |
| 模型 | deepseek-v4-pro（主力）/ v4-flash（轻量） | 推理 + 快速任务分层 |
| 配置存储 | `~/.xiaoli/config.json` | 简单、跨平台 |
| 会话状态 | SQLite（本地） | 持久记忆、断电不丢 |
| 发布 | npm registry | `npm publish` |

---

## 2. 安装体验

```bash
# 安装
npm install -g xiaoli

# 配置 DeepSeek API key
xiaoli config set api-key sk-xxx

# 开始使用
xiaoli "帮我分析这个项目的代码结构"
```

**零配置启动**：如果未设置 API key，首次运行自动引导配置。

## 5. 核心模块详解

### 5.1 对话引擎

```
对话引擎职责：
  1. 接收用户任务描述
  2. 调用 AI 生成提问列表（5-15 个问题）
  3. 逐个展示问题，收集用户回答
  4. 全部回答后 → 调用 AI 生成执行计划
  5. 用户确认 → 将计划转为步骤列表 → 交给执行器

状态机：
  IDLE → QUESTIONING → WAITING_ANSWER → ... → PLAN_READY
       → WAITING_CONFIRM → EXECUTING → COMPLETED / STUCK
```

### 5.2 执行器

```
执行器职责：
  1. 按步骤顺序执行计划
  2. 每步执行前：检查安全模块
  3. 每步执行后：记录结果到 SQLite
  4. 遇到环境问题：尝试自动修复（最多 3 次）
  5. 无法自动修复：暂停并标记 STUCK，写入解决尝试

环境自愈策略：
  - JDK 版本问题 → 检测已安装版本，调整 JAVA_HOME
  - npm/pip 依赖冲突 → 尝试 --legacy-peer-deps / 降级
  - 数据库连接失败 → 检查配置文件，尝试备选连接
  - 端口占用 → kill 旧进程，重试
```

### 5.3 安全模块

```
安全模块职责：
  1. 每步执行前扫描命令/操作
  2. 匹配危险模式 → 拦截并询问用户
  3. 匹配安全模式 → 放行
  4. 匹配需确认模式（计划外操作） → 询问用户

危险模式库（持续扩展）：
  SHELL: "rm -rf" / "sudo rm" / "chmod 777" / "> /dev/sda"
  SQL: "DROP TABLE" / "DROP DATABASE" / "TRUNCATE"
  GIT: "push --force" / "hard reset" / "push --delete"
  FILE: 删除 >100行 或 删除 >5个文件 同时操作
  ENV: 修改 ~/.bashrc / /etc/hosts / 生产配置
```

### 5.4 AI 模块

```
AI 模块职责：
  1. 任务解析 → 提问生成（deepseek-v4-pro，高推理）
  2. 计划生成（deepseek-v4-pro，高推理）
  3. 代码分析 / 错误诊断（deepseek-v4-pro）
  4. 轻量操作决策（deepseek-v4-flash，低延迟）
  5. Token 使用统计和管理

分流策略：
  ┌─────────────────────┬──────────────┐
  │ 任务                  │ 模型          │
  ├─────────────────────┼──────────────┤
  │ 提问生成              │ v4-pro       │
  │ 计划生成              │ v4-pro       │
  │ 代码分析 / 错误诊断    │ v4-pro       │
  │ 简单文件操作决策       │ v4-flash     │
  │ Git commit message   │ v4-flash     │
  │ 步骤摘要 / 进度汇报    │ v4-flash     │
  └─────────────────────┴──────────────┘
```

## 6. 数据存储

### 6.1 SQLite 表设计

```sql
-- 任务表
CREATE TABLE tasks (
  id TEXT PRIMARY KEY,
  description TEXT NOT NULL,       -- 原始任务描述
  status TEXT NOT NULL,            -- questioning/planning/executing/completed/failed
  plan TEXT,                       -- 执行计划 JSON
  created_at INTEGER,
  updated_at INTEGER
);

-- 问答记录表
CREATE TABLE qa_exchanges (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  task_id TEXT NOT NULL,
  question TEXT NOT NULL,          -- xiaoli 提的问题
  answer TEXT,                     -- 用户的回答
  question_order INTEGER NOT NULL,
  created_at INTEGER,
  FOREIGN KEY (task_id) REFERENCES tasks(id)
);

-- 执行步骤表
CREATE TABLE task_steps (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  task_id TEXT NOT NULL,
  step_order INTEGER NOT NULL,
  description TEXT NOT NULL,       -- 步骤描述
  status TEXT NOT NULL,            -- pending/running/completed/failed/skipped
  result TEXT,                     -- 执行结果
  error TEXT,                      -- 错误信息
  recovery_attempts INTEGER DEFAULT 0,
  started_at INTEGER,
  completed_at INTEGER,
  FOREIGN KEY (task_id) REFERENCES tasks(id)
);

-- 项目记忆表
CREATE TABLE project_memory (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  project_path TEXT NOT NULL,
  key TEXT NOT NULL,               -- 如 "tech_stack", "db_version"
  value TEXT NOT NULL,
  updated_at INTEGER,
  UNIQUE(project_path, key)
);

-- 用户偏好表
CREATE TABLE user_preferences (
  key TEXT PRIMARY KEY,
  value TEXT NOT NULL,
  updated_at INTEGER
);

-- 操作日志表（审计用）
CREATE TABLE audit_log (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  task_id TEXT,
  step_id INTEGER,
  operation TEXT NOT NULL,         -- shell/sql/file/git
  command TEXT NOT NULL,           -- 完整命令
  result TEXT,                     -- 执行结果摘要
  created_at INTEGER,
  FOREIGN KEY (task_id) REFERENCES tasks(id)
);
```
